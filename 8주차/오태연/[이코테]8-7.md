### 문제풀이 핵심아이디어 또는 새롭게 알게된 내용: 
    Bottom up 방식이용해서 작은 단위에서부터 차근차근 답을 도출
    
### 코드의 시간복잡도와 그 이유:
    O(N): 반복문 N-2번 돌기 때문
    
### 주석이 가득 담긴 코드:
```python
# n 입력받기
n = int(input())

# dp 테이블 초기화
d = [0]*1001

# Bottom-up
d[1] = 1
d[2] = 3
for i in range(3,n+1):
    d[i]=(d[i-1]+2*d[i-2]) % 796796

# 계산결과 출력
print(d[n])

```
